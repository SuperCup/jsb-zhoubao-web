# 发布流程

本项目采用“测试先行、正式发布需群内确认”的发布规则。

## 发布地址

- 测试地址：`https://supercup.github.io/jsb-zhoubao-web/`
- 正式地址：`https://agent.ismartgo.com/carlsberg/weekly`

## 默认发布规则

- 所有代码、数据和看板展示修改，默认只发布到测试地址。
- 推送 `main` 后，GitHub Actions workflow `Deploy Test GitHub Pages` 会自动构建静态包，并更新 `gh-pages` 分支。
- `gh-pages` 分支只作为测试发布源使用，不代表正式地址已更新。
- 未经群成员验证通过并明确允许，不得更新正式地址。

## 正式发布门禁

正式地址发布必须同时满足：

1. 测试地址已完成发布并可访问。
2. 群成员已在群聊中完成验证。
3. 群成员明确回复允许发布到正式地址。

满足以上条件后，才允许通过 `agent.ismartgo.com` 管理后台或对应 API 发布到正式地址。

## 操作清单

1. 修改代码或数据。
2. 本地验证：至少运行 `npm run lint`、`npm run build`、`npm run build:github`。
3. 推送 `main`，等待测试地址更新。
4. 在群聊中发送测试地址和变更说明，等待群成员验证。
5. 收到明确允许后，再发布正式地址。
6. 发布正式地址后，在群聊中回报正式地址、发布时间和对应提交。

## 禁止事项

- 不得把正式发布作为默认动作。
- 不得在群成员验证前发布正式地址。
- 不得把 `agent.ismartgo.com` 的账号、密码、token 或其他密钥写入仓库。
